# Rider 2025.3 GitHub Copilot MCP Setup - Complete Guide

## TL;DR - The Critical Missing Piece

**The `mcpConfigPath` option in `github-copilot.xml` is REQUIRED** for GitHub Copilot to find your MCP servers!

Without it, GitHub Copilot has NO IDEA where to look for MCP configurations.

---

## Complete File Checklist

Here are ALL the files that need to exist for MCP to work in Rider 2025.3 with GitHub Copilot:

### ✅ 1. Bridge Script
**Location:** `~/.local/bin/mcp-http-bridge`  
**Size:** ~2 KB  
**Permissions:** Executable (`chmod +x`)  
**Purpose:** Translates MCP stdio protocol to HTTP calls to the remote server

### ✅ 2. Wrapper Script
**Location:** `~/.local/bin/mcp-agency-agents`  
**Size:** ~122 bytes  
**Permissions:** Executable (`chmod +x`)  
**Content:**
```bash
#!/bin/bash
export MCP_URL="https://agency-agents-mcp.regtransfers.dev"
exec ~/.local/bin/mcp-http-bridge "$@"
```

### ✅ 3. GitHub Copilot Plugin Config ⚠️ **CRITICAL**
**Location:** `~/.config/JetBrains/Rider2025.3/options/github-copilot.xml`  
**MUST contain:**
```xml
<application>
  <component name="github-copilot">
    <option name="mcpConfigPath" value="/home/YOUR_USERNAME/.config/github-copilot/intellij/mcp.json" />
    <option name="signinNotificationShown" value="true" />
    <option name="terminalRulesVersion" value="1" />
    <mcpSamplingAllowedModels>
      <option value="claude-sonnet-4.5" />
      <option value="gpt-4.1" />
      <option value="gpt-5.4" />
      <option value="gpt-5.4-mini" />
    </mcpSamplingAllowedModels>
  </component>
</application>
```

**⚠️ WARNING:** This file is auto-generated by Rider. You MUST close Rider before editing it, or your changes will be overwritten!

### ✅ 4. GitHub Copilot MCP Server Definition
**Location:** `~/.config/github-copilot/intellij/mcp.json`  
**Content:**
```json
{
    "servers": {
        "agency-agents": {
            "type": "stdio",
            "command": "/home/YOUR_USERNAME/.local/bin/mcp-agency-agents"
        }
    }
}
```

### ✅ 5. Rider MCP Config (Optional - for Rider's built-in LLM features)
**Location:** `~/.config/JetBrains/Rider2025.3/mcp_config.json`  
**Content:**
```json
{
  "mcpServers": {
    "agency-agents": {
      "command": "/home/YOUR_USERNAME/.local/bin/mcp-agency-agents",
      "args": []
    }
  }
}
```

### ✅ 6. Rider MCP Server Status (Auto-generated by Rider)
**Location:** `~/.config/JetBrains/Rider2025.3/options/llm.mcpServers.xml`  
**Should show (after first Rider launch):**
```xml
<option name="enabled" value="true" />
<option name="name" value="agency-agents" />
<option name="command" value="/home/YOUR_USERNAME/.local/bin/mcp-agency-agents" />
```

## Installation Script (Copy-Paste This)

**⚠️ Close Rider before running this!**

```bash
#!/bin/bash

# Replace YOUR_USERNAME with your actual username!
USERNAME=$(whoami)
RIDER_VERSION="Rider2025.3"

echo "🚀 Setting up Agency Agents MCP for Rider 2025.3 + GitHub Copilot"
echo ""

# Step 1: Download bridge script
echo "📥 Step 1/5: Downloading bridge script..."
mkdir -p ~/.local/bin
curl -o ~/.local/bin/mcp-http-bridge https://raw.githubusercontent.com/Regtransfers/agency-agents-mcp/main/mcp-http-bridge.mjs
chmod +x ~/.local/bin/mcp-http-bridge
echo "✅ Bridge downloaded to ~/.local/bin/mcp-http-bridge"

# Step 2: Create wrapper script
echo "📝 Step 2/5: Creating wrapper script..."
cat > ~/.local/bin/mcp-agency-agents << 'EOF'
#!/bin/bash
export MCP_URL="https://agency-agents-mcp.regtransfers.dev"
exec ~/.local/bin/mcp-http-bridge "$@"
EOF
chmod +x ~/.local/bin/mcp-agency-agents
echo "✅ Wrapper created at ~/.local/bin/mcp-agency-agents"

# Step 3: Create GitHub Copilot MCP server definition
echo "⚙️ Step 3/5: Creating GitHub Copilot MCP config..."
mkdir -p ~/.config/github-copilot/intellij
cat > ~/.config/github-copilot/intellij/mcp.json << EOF
{
    "servers": {
        "agency-agents": {
            "type": "stdio",
            "command": "/home/$USERNAME/.local/bin/mcp-agency-agents"
        }
    }
}
EOF
echo "✅ MCP server definition created at ~/.config/github-copilot/intellij/mcp.json"

# Step 4: Configure GitHub Copilot plugin (CRITICAL!)
echo "🔧 Step 4/5: Configuring GitHub Copilot plugin..."
mkdir -p ~/.config/JetBrains/$RIDER_VERSION/options
cat > ~/.config/JetBrains/$RIDER_VERSION/options/github-copilot.xml << EOF
<application>
  <component name="github-copilot">
    <option name="mcpConfigPath" value="/home/$USERNAME/.config/github-copilot/intellij/mcp.json" />
    <option name="signinNotificationShown" value="true" />
    <option name="terminalRulesVersion" value="1" />
    <mcpSamplingAllowedModels>
      <option value="claude-sonnet-4.5" />
      <option value="gpt-4.1" />
      <option value="gpt-5.4" />
      <option value="gpt-5.4-mini" />
    </mcpSamplingAllowedModels>
  </component>
</application>
EOF
echo "✅ GitHub Copilot config created at ~/.config/JetBrains/$RIDER_VERSION/options/github-copilot.xml"

# Step 5: Test the setup
echo "🧪 Step 5/5: Testing MCP connection..."
RESULT=$(~/.local/bin/mcp-agency-agents <<< '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' 2>&1)
if echo "$RESULT" | grep -q "list_agents"; then
    echo "✅ MCP server is working!"
    echo "   Tools available: $(echo "$RESULT" | grep -o '"name":"[^"]*"' | wc -l)"
else
    echo "❌ MCP server test failed. Output:"
    echo "$RESULT"
    exit 1
fi

echo ""
echo "🎉 Setup complete!"
echo ""
echo "📋 Next steps:"
echo "   1. Launch Rider 2025.3"
echo "   2. Open GitHub Copilot Chat"
echo "   3. Type: 'list available agents'"
echo "   4. You should see 144+ agent personas!"
echo ""
echo "💡 Tip: If tools don't appear, start a NEW Copilot conversation (+ button)"
```

**Save this as `install-mcp.sh`, make it executable, and run it:**
```bash
chmod +x install-mcp.sh
./install-mcp.sh
```

## Why This Is So Complicated

GitHub Copilot's MCP implementation in Rider 2025.3 requires **3 separate config mechanisms**:

1. **`github-copilot.xml`** - Tells the GitHub Copilot plugin where to find MCP configs
2. **`mcp.json`** - Defines what MCP servers exist and how to start them
3. **Wrapper script** - Sets environment variables (since `mcp.json` doesn't support `env` in Rider)

All three MUST be present and configured correctly, or it silently fails with no error messages.

---

## Common Mistakes

### ❌ Mistake #1: Editing `github-copilot.xml` while Rider is running
**Result:** Rider overwrites your changes on exit  
**Fix:** Always close Rider before editing this file

### ❌ Mistake #2: Forgetting to add `mcpConfigPath`
**Result:** GitHub Copilot can't find your MCP configs, shows 0 tools  
**Fix:** Add the `<option name="mcpConfigPath" ... />` line to `github-copilot.xml`

### ❌ Mistake #3: Using relative paths or `$HOME` in configs
**Result:** Rider/Copilot can't find the scripts  
**Fix:** Use absolute paths like `/home/yourusername/.local/bin/...`

### ❌ Mistake #4: Not restarting Rider after config changes
**Result:** Old configuration still loaded  
**Fix:** Fully close and reopen Rider (not just restart project)

### ❌ Mistake #5: Testing in an old Copilot conversation
**Result:** MCP tools not available in that conversation  
**Fix:** Click the "+" button to start a fresh conversation

## What Each File Does

| File | Purpose | Required For |
|------|---------|-------------|
| `mcp-http-bridge` | Bridges MCP stdio ↔ HTTP to remote server | All setups |
| `mcp-agency-agents` | Sets `MCP_URL` env var and calls bridge | Rider 2025.3+ |
| `github-copilot.xml` | **Tells Copilot WHERE to find MCP configs** | Rider 2025.3+ GitHub Copilot |
| `intellij/mcp.json` | Defines the MCP server | Rider 2025.3+ GitHub Copilot |
| `mcp_config.json` | Rider's built-in LLM features (not Copilot) | Optional |
| `llm.mcpServers.xml` | Auto-generated status file | Auto-created by Rider |

## Testing Without Rider

You can test the MCP bridge works before even launching Rider:

```bash
# Should return JSON with 9 tools
~/.local/bin/mcp-agency-agents <<< '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

# Should return agent data
~/.local/bin/mcp-agency-agents <<< '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"list_agents","arguments":{}}}'
```

If these commands fail, the MCP server connection is broken and needs fixing before Rider will work.
