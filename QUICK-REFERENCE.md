## 📋 Top 10 Skills for Your Team

### 1. **brainstorming** - Design before coding
**When:** Starting any new feature or major change
```
Use brainstorming skill to design an authentication system with JWT tokens
```

### 2. **test-driven-development** - TDD workflow
**When:** Building features that need solid test coverage
```
Activate TDD skill and help me add payment processing with Stripe API
```

### 3. **api-design-principles** - REST/GraphQL design
**When:** Creating new APIs or refactoring existing endpoints
```
Use api-design-principles to design a REST API for inventory management
```

### 4. **code-review-excellence** - Thorough reviews
**When:** Before merging any PR
```
Activate code-review-excellence and review this authentication PR
```

### 5. **systematic-debugging** - Methodical debugging
**When:** Tracking down complex bugs
```
Use systematic-debugging to trace this race condition in the queue processor
```

### 6. **e2e-testing** - End-to-end tests
**When:** Testing complete user workflows
```
Use e2e-testing skill to create Playwright tests for our checkout flow
```

### 7. **api-security-testing** - API vulnerability testing
**When:** Security audits, before production releases
```
Activate api-security-testing and audit our authentication endpoints
```

### 8. **kubernetes-deployment** - K8s + Flux patterns
**When:** Deploying services, creating manifests
```
Use kubernetes-deployment to create a deployment for our new microservice
```

### 9. **gitops-workflow** - GitOps and Flux CD
**When:** Setting up infrastructure automation
```
Use gitops-workflow to set up Flux CD for staging environment
```

### 10. **ci-cd-automation** - Pipeline workflows
**When:** Automating builds, tests, deployments
```
Use ci-cd-automation to create a GitHub Actions workflow for API tests
```

## 🎓 Learning Path

### Week 1: Start with the basics
- Try `brainstorming` for any new feature
- Use `test-driven-development` on one feature
- Practice `code-review-excellence` on PRs

### Week 2: Level up
- Apply `api-design-principles` to an API
- Try `systematic-debugging` on a tricky bug
- Use `e2e-testing` for critical workflows

### Week 3: Infrastructure
- Deploy with `kubernetes-deployment`
- Set up `gitops-workflow`
- Automate with `ci-cd-automation`

### Week 4: Security & Quality
- Run `api-security-testing` audits
- Combine agents + skills for complex tasks
- Create your own team workflows

## ⚡ Pro Tips

1. **Use skills for methodology, agents for expertise**
   - Skill = HOW to work (TDD, API design)
   - Agent = WHO you are (Backend Architect, Security Engineer)

2. **Combine for best results**
   ```
   "Activate [agent] + use [skill1] and [skill2] skills"
   ```

3. **Start specific, then explore**
   ```
   First: "Use test-driven-development skill"
   Later: "Search for skills about testing" to discover more
   ```

4. **Document your team's favorites**
   - Track which combinations work best
   - Share workflows in team docs
   - Build your own best practices

5. **One chat = one context**
   - Start fresh chat for new topics
   - Prevents conflicting skills/agents
   - Cleaner conversations
